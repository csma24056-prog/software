# 英語継続アプリ テストケース定義書 (Test Cases for English Habit App)

# 📋 テストケース一覧 (全24ケース)

# 1. ユーザー登録・目標設定（4ケース）


| ID | テスト対象機能 | テスト内容（操作手順） | 期待される結果（正解挙動） |
| :--- | :--- | :--- | :--- |
| **TC-01** | 新規会員登録 | メールアドレス、またはSNS連携（Google/Apple）でアカウントを作成する。 | 正常に登録され、初期設定（オンボーディング）画面へ遷移すること。 |
| **TC-02** | プロフィール編集 | マイページから「ニックネーム」や「アイコン画像」を変更する。 | 変更内容が即座に保存され、アプリ内の各画面に正しく反映されること。 |
| **TC-03** | 学習目標の設定 | 1日の目標学習時間（例：15分）や、学習目的（例：TOEIC、日常会話）を設定・変更する。 | 設定した目標に合わせた学習プランやダッシュボードの表示に切り替わること。 |
| **TC-04** | 登録キャンセルの挙動 | 会員登録の途中でアプリを閉じ、再度アプリを起動する。 | 登録が中途半端な状態でログイン状態にならず、再度登録画面（または初期画面）からスタートすること。 |

# 2. メイン学習機能・データ保存（5ケース）

| ID | テスト対象機能 | テスト内容（操作手順） | 期待される結果（正解挙動） |
| :--- | :--- | :--- | :--- |
| **TC-05** | レッスン再生（音声） | リスニング問題や単語クイズで音声再生ボタンをタップする。 | 音声が遅延なくクリアに再生され、バックグラウンド再生や音量調節が連動すること。 |
| **TC-06** | クイズの正誤判定 | 4択クイズやタイピング問題で、正解・不正解をそれぞれ入力する。 | 正しいアニメーション/効果音が鳴り、解説画面が適切に表示されること。 |
| **TC-07** | レッスン完了と記録 | 1つのレッスンを最後まで解き進め、完了ボタンをタップする。 | 「本日の学習時間」としてカウントされ、進捗メーターが正しく進むこと。 |
| **TC-08** | お気に入り/復習帳登録 | 間違えた単語や、気になるフレーズを「お気に入り」に登録する。 | 復習専用のフォルダやタブにその項目が追加され、後からソート・確認できること。 |
| **TC-09** | バックグラウンド移行 | レッスンの回答途中で、別のアプリを開いたりホーム画面に戻ったりしてから、再度アプリを開く。 | レッスンが強制終了せず、中断した直前の設問から再開できること。 |

# 3. 継続・モチベーション（ストリーク・報酬）（6ケース）

| ID | テスト対象機能 | テスト内容（操作手順） | 期待される結果（正解挙動） |
| :--- | :--- | :--- | :--- |
| **TC-10** | ストリーク加算 | 前日に学習を完了している状態で、当日の目標（レッスン）を完了させる。 | 連続学習日数（ストリーク）のカウントが「+1」されること。 |
| **TC-11** | ストリーク未達成（リセット）| 1日中アプリを開かず学習をスキップし、翌日にアプリを起動する。 | ストリークが「0」にリセットされること。 |
| **TC-12** | ストリーク補正（救済） | スキップアイテム（または有料チケットなど）を使用して、途切れたストリークを復活させる。 | ストリークが途切れる前の日数に正しく復元されること。 |
| **TC-13** | 習慣化カレンダー | 月間カレンダー画面を開き、学習した日とそうでない日の表示を確認する。 | 学習完了日にはスタンプや色がつき、未完了日と明確に視覚的区別ができること。 |
| **TC-14** | ミッション・バッジ達成 | 「7日間連続学習」「累計100単語突破」などの条件を満たす。 | 条件達成のポップアップ（演出）が表示され、バッジ一覧画面でロックが解除されること。 |
| **TC-15** | グラフの週次/月次切り替え | 学習統計画面で、グラフの表示を「週」「月」「年」に切り替える。 | それぞれの期間に応じた累計学習時間や平均学習時間が、正確に計算されてグラフ表示されること。 |

# 4. 通知・リマインダー（4ケース）


| ID | テスト対象機能 | テスト内容（操作手順） | 期待される結果（正解挙動） |
| :--- | :--- | :--- | :--- |
| **TC-16** | 指定時刻のプッシュ通知 | ユーザーが設定した時刻（例：毎日20:00）に、端末にプッシュ通知が届くか確認する。 | 設定通りの時刻に、学習を促す文言のプッシュ通知が表示されること。 |
| **TC-17** | 未学習状態の警告通知 | 23:00時点でその日の学習が未完了の場合に、夜間のリマインド通知が届くか確認する。 | 日付が変わる前に、ストリーク維持を促す緊急通知が届くこと。 |
| **TC-18** | 通知からの画面遷移 | 届いたプッシュ通知をタップしてアプリを起動する。 | トップ画面ではなく、直接「本日の学習開始画面」や「前回の続き」へディープリンク（遷移）すること。 |
| **TC-19** | 通知拒否時の挙動 | 端末の設定でアプリの通知を「許可しない」に変更する。 | アプリ内で通知エラーによるクラッシュが発生せず、通知も一切届かなくなること。 |

# 5. 異常系・エッジケース（5ケース）


| ID | テスト対象機能 | テスト内容（操作手順） | 期待される結果（正解挙動） |
| :--- | :--- | :--- | :--- |
| **TC-20** | 通信遮断（オフライン） | 通信を遮断（機内モード）した状態でアプリを起動し、学習を行う。 | オフライン用のエラー画面が出るか、もしくはローカル保存モードで学習が継続できること（仕様に準ずる）。 |
| **TC-21** | 通信復帰時の同期 | オフラインで行った学習データを保持したまま通信を再開（Wi-Fi接続など）する。 | バックグラウンドでサーバーへデータが送信され、学習履歴やストリークが最新状態に同期されること。 |
| **TC-22** | 日付を跨ぐ学習 | 23:59にレッスンを開始し、翌0:05にレッスンを完了させる。 | アプリの定義（「開始した日の実績」か「完了した日の実績」か）に従い、データが矛盾なく処理されること。 |
| **TC-23** | タイムゾーン跨ぎ | 日本で学習後、端末のタイムゾーンを海外（例：ロンドン）に変更してアプリを開く。 | 時差によってストリークの計算がバグを起こさず、変更後の現地時間に基づいて判定されること。 |
| **TC-24** | ストレージ容量不足 | 端末の空き容量が極端に少ない状態で、新しい音声教材データをダウンロードしようとする。 | アプリが強制終了（クラッシュ）せず、「容量が不足しています」という適切なエラーメッセージが表示されること。 |
